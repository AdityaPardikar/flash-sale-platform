Installation Options

‚≠ê Option 1: Docker Desktop (Recommended)

This is the easiest and most consistent way to set up your database.

  1. Download Docker Desktop for Windows from docker.com
  2. Install and restart your computer
  3. Run: docker compose up -d in the project directory
  4. Run migrations: npm run migrate --workspace=backend
  5. Run seed data: npm run seed --workspace=backend
‚úì Advantages: Isolated environment, easy cleanup, reproducible setup

üíª Option 2: Local PostgreSQL Installation

Install PostgreSQL directly on your system.

  1. Install PostgreSQL 16 from postgresql.org
  2. During installation, set password to flash_password
  3. Create database: createdb -U postgres flash_sale_db
  4. Create user with SQL:
    CREATE USER flash_user WITH PASSWORD 'flash_password'; GRANT ALL PRIVILEGES ON DATABASE flash_sale_db TO flash_user;
  5. Update .env file in backend with your PostgreSQL connection details
  6. Run migrations: npm run migrate --workspace=backend
  7. Run seed data: npm run seed --workspace=backend
‚úì Advantages: Full control, persistent data, no Docker required

‚òÅÔ∏è Option 3: Cloud PostgreSQL

Use a managed PostgreSQL service from a cloud provider.

  1. Create a free PostgreSQL instance on any cloud provider:
    • ElephantSQL
    • Supabase
    • DigitalOcean
  2. Get the connection string from the provider
  3. Update .env file in backend:
    DATABASE_URL=postgresql://user:pass@host:port/dbname
  4. Run migrations: npm run migrate --workspace=backend
  5. Run seed data: npm run seed --workspace=backend
‚úì Advantages: No local setup, managed backups, accessible from anywhere

üöÄ Getting Started

Step-by-Step Setup

  1. Complete one of the installation options above
  2. Verify PostgreSQL is running on localhost:5432 (default port)
  3. Run migrations: npm run migrate --workspace=backend
  4. Add test data: npm run seed --workspace=backend
  5. Start the API server: npm run dev --workspace=backend
  6. Test endpoints at http://localhost:3000

üêõ Troubleshooting

Connection Error? Make sure PostgreSQL is running and the connection string in your .env file is correct.
Migration Failed? Check that all database tables don't already exist, or run DROP DATABASE flash_sale_db first.
No Test Data? Run npm run seed --workspace=backend after migrations complete.
